# manual-git
Explicación comandos git
git init: Inicializa un nuevo repositorio Git en el directorio actual. Crea la carpeta .git que contiene toda la información del control de versiones.
git clone: Clona un repositorio remoto en tu máquina local. Copia todo el historial y archivos del proyecto.
git add: Añade archivos al área de preparación (staging area), indicando que están listos para ser confirmados.
git commit	
git log:	Muestra el historial de commits realizados, incluyendo autor, fecha y mensaje.
git checkout:	Permite cambiar entre ramas o restaurar archivos a versiones anteriores. También se usa para crear ramas si se combina con -b.
git branch:	Lista las ramas existentes o crea una nueva rama si se usa con un nombre adicional.
git push:	Envía los commits locales al repositorio remoto, actualizando su contenido.
git pull:	Descarga los cambios del repositorio remoto y los fusiona con la rama actual.
git merge:	Fusiona el contenido de una rama con otra, integrando los cambios en la rama activa.
