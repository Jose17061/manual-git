# manual-git
## Indice de comandos
1. `Git init`
2. `Git clone`
3. `Git add`
4. `Git commit`
5. `Git log`
6. `Git checkout`
7. `Git branch`
8. `Git push`
9. `Git pull`
10. `Git merge`

## Git init
**Descripción:**
Inicializa un nuevo repositorio Git en el directorio actual. Crea la carpeta .git que contiene toda la información del control de versiones.
## Git clone: 
**Descripción:**
Clona un repositorio remoto en tu máquina local. Copia todo el historial y archivos del proyecto.
## Git add: 
**Descripción:**
Añade archivos al área de preparación (staging area), indicando que están listos para ser confirmados.
## Git commit	
**Descripción:**
Guarda los cambios añadidos al área de preparación en el historial del repositorio. Requiere un mensaje descriptivo.
## Git log:	
**Descripción:**
Muestra el historial de commits realizados, incluyendo autor, fecha y mensaje.
## Git checkout:	
**Descripción:**
Permite cambiar entre ramas o restaurar archivos a versiones anteriores. También se usa para crear ramas si se combina con -b.
## Git branch:
**Descripción:**
Lista las ramas existentes o crea una nueva rama si se usa con un nombre adicional.
## Git push:
**Descripción:**
Envía los commits locales al repositorio remoto, actualizando su contenido.
## Git pull:
**Descripción:**
Descarga los cambios del repositorio remoto y los fusiona con la rama actual.
## Git merge:
**Descripción:**
Fusiona el contenido de una rama con otra, integrando los cambios en la rama activa.
